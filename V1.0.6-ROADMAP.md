# Braya Browser v1.0.6 Roadmap

## Pillars
1. **Icon & branding polish** – ensure the browser icon sizes play nicely in the GNOME shell bar and toolbars, then refresh the themes/slide bars so the chrome feels deliberate.
2. **Password manager security & UX** – finalize the master-password workflow, autofill helpers/toasts, multi-step capture configuration, and exception handling so the vault behaves more like Bitwarden.
3. **Settings & privacy redesign** – evolve General/Security/Privacy panels (renewed cards, rail navigation, rich controls for cookies and site permissions) to match the more polished experience users expect.

## Progress
- ✅ Phase 1 password manager work landed: libsodium/Argon2 migration, master-salt persistence, reusable modals, and migration-aware loaders are documented in `PASSWORD_MANAGER_PLAN.md` and `docs/technical/QUICK_SUMMARY.md`.
- ✅ Settings now rebuild the “Manage Site Permissions” list with working “Clear” buttons that delete the selected `WebKitWebsiteData` entry via `webkit_network_session_get_website_data_manager`.
- ✅ Multi-step login capture toggle now lives inside a `createSettingsRow` entry, keeping it aligned with the other switches and reducing its footprint.

## In progress
- Expand Privacy → Cookies & Site Data with filtering, per-domain delete actions, and a clear/revoke flow that ties into the new permissions list.
- Harden password manager metadata/strength reporting, autofill/toast behavior, and "Never save for this site" overrides so the vault feels more complete.
- Continue the top/side-bar redesign for Settings so the panels (General, Security, Passwords) look intentional, with cohesive slide bars and cards.

## Next milestones
1. Add an enable/disable selector for the multi-step capture path, plus inline help text and per-site overrides.
2. Build a Cookie Manager view that exposes per-site cookie counts and lets users delete or whisk entire groups.
3. Finish the Password Manager master password change/reset dialogs with recovery instructions and toasts that highlight success/failure.
4. Sketch the first pieces of a standalone password vault app once the in-browser UX feels stable.

## Future/backlog ideas
- Dedicated password manager app on GTK and/or mobile.
- Multi-step login automation hints (autofill prompts for multi-screen flows).
- Syncing vault entries with an encrypted cloud backend in a future phase.
