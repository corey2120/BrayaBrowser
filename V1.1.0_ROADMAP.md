## 🔥 CRITICAL PRIORITY - Must Fix

### 1. Favicon System Overhaul ⭐⭐⭐ CRITICAL

**Current Problems:**
- Favicons don't load reliably
- Race conditions cause missing icons
- Cache system doesn't work properly
- Tabs show generic icons instead of site favicons
- Bookmarks lose favicons

**Root Causes:**
- Async loading without proper callbacks
- No retry mechanism for failed downloads
- Cache corruption issues
- WebView destruction during favicon fetch
- No fallback to default favicon locations

**Complete Solution:**

#### Phase 1: New Favicon Manager Class (4-6 hours)

Create `src/BrayaFaviconManager.cpp` with:

```cpp
class BrayaFaviconManager {
public:
    // Singleton instance
    static BrayaFaviconManager& getInstance();

    // Request favicon with callback
    void requestFavicon(const std::string& url,
                       std::function<void(GdkTexture*)> callback,
                       GtkWidget* tab = nullptr);

    // Synchronous get from cache
    GdkTexture* getCachedFavicon(const std::string& url);

    // Pre-load favicons for multiple URLs
    void preloadFavicons(const std::vector<std::string>& urls);

    // Clear cache
    void clearCache();
    void clearExpiredCache(int days = 7);

private:
    // Smart cache with domain-based storage
    std::unordered_map<std::string, FaviconEntry> cache;

    // Download queue to prevent duplicate requests
    std::unordered_map<std::string, std::vector<std::function<void(GdkTexture*)>>> pendingRequests;

    // Favicon download with multiple fallbacks
    void downloadFavicon(const std::string& url);
    void tryFaviconSource(const std::string& baseUrl, int attempt);

    // Sources to try (in order):
    // 1. Link rel="icon" from HTML
    // 2. /favicon.ico
    // 3. /favicon.png
    // 4. Google favicon service as last resort

    // Persistent cache
    void saveCacheToDisk();
    void loadCacheFromDisk();

    struct FaviconEntry {
        GdkTexture* texture;
        std::string url;
        time_t fetchedAt;
        bool isDefault; // Generic icon vs real favicon
    };
};
```

#### Phase 2: Reliable Download System (2-3 hours)

```cpp
void BrayaFaviconManager::downloadFavicon(const std::string& pageUrl) {
    std::string domain = extractDomain(pageUrl);

    // Check if already downloading
    if (pendingRequests.find(domain) != pendingRequests.end()) {
        return; // Queue callback will be notified
    }

    // Try sources in order:
    std::vector<std::string> sources = {
        pageUrl, // Let WebKit extract favicon from HTML
        "https://" + domain + "/favicon.ico",
        "https://" + domain + "/favicon.png",
        "https://" + domain + "/apple-touch-icon.png",
        "https://www.google.com/s2/favicons?domain=" + domain
    };

    tryNextSource(domain, sources, 0);
}

void BrayaFaviconManager::tryNextSource(
    const std::string& domain,
    const std::vector<std::string>& sources,
    int currentIndex) {

    if (currentIndex >= sources.size()) {
        // All sources failed, use default icon
        notifyCallbacks(domain, getDefaultFavicon());
        return;
    }

    // Try current source with timeout
    WebKitWebView* helperView = createHelperWebView();

    g_signal_connect(helperView, "favicon-changed",
        G_CALLBACK(+[](WebKitWebView* view, GParamSpec* pspec, gpointer data) {
            // Got favicon, save to cache
            // Notify all pending callbacks
            // Destroy helper view
        }), this);

    // Load URL with 5 second timeout
    webkit_web_view_load_uri(helperView, sources[currentIndex].c_str());

    // Set timeout to try next source
    g_timeout_add_seconds(5, G_SOURCE_FUNC(+[](gpointer data) -> gboolean {
        // Timeout reached, try next source
        auto* ctx = static_cast<DownloadContext*>(data);
        ctx->manager->tryNextSource(ctx->domain, ctx->sources, ctx->index + 1);
        return G_SOURCE_REMOVE;
    }), new DownloadContext{this, domain, sources, currentIndex});
}
```

**Previous Attempt Issues:**

**New Approach - Learn from Previous Mistakes:**

**New Implementation Strategy:**

#### Phase 1: Simple Backend First (3-4 hours)

**Keep It Simple:**
- One folder = one saved state
- No nested folders
- No complex hierarchies
- Just save and restore

#### Phase 5: Auto-Save Last Session (1 hour)

```cpp
// On browser close
void BrayaWindow::onCloseRequest() {
    // Auto-save current state
    BrayaTabFolder autoSave = BrayaTabFolder::saveCurrentTabs(this, "__autosave__");
    autoSave.saveToDisk("~/.config/braya-browser/sessions/autosave.json");
}

// On browser start
void BrayaWindow::onStartup() {
    // Check if autosave exists
    if (fs::exists("~/.config/braya-browser/sessions/autosave.json")) {
        GtkWidget* dialog = gtk_message_dialog_new(
            GTK_WINDOW(window),
            GTK_DIALOG_MODAL,
            GTK_MESSAGE_QUESTION,
            GTK_BUTTONS_YES_NO,
            "Restore Previous Session?"
        );

        int response = gtk_dialog_run(GTK_DIALOG(dialog));
        if (response == GTK_RESPONSE_YES) {
            BrayaTabFolder::fromFile("autosave.json").restoreTabs(this, true);
        }

        gtk_widget_destroy(dialog);
    }
}
```

## 🐛 Bug Fixes

### 3. Fix Deprecation Warnings ⭐ HIGH

**Current Warnings:**
- GtkComboBoxText → GtkDropDown
- GtkColorButton → GtkColorDialog
- GtkFileChooserDialog → GtkFileDialog
- gtk_widget_show → gtk_widget_set_visible

**Impact:** ~50 deprecation warnings on every build

**Solution:** Migrate to GTK4 modern APIs

**Time Estimate:** 4-6 hours

---

### 4. Memory Leak Fixes ⭐ MEDIUM

**Current Issues:**
- WebView cleanup not always complete
- Favicon textures not freed
- History entries accumulate without limit

**Solutions:**
- Proper GObjectPtr usage everywhere
- Weak references where appropriate
- History limit (default 1000 entries)

**Time Estimate:** 3-4 hours

## ✅ Success Criteria

v1.1.0 will be considered successful if:

- ✅ Favicons load reliably 95%+ of the time
- ✅ Favicons persist across browser restarts
- ✅ Tab sessions save and restore correctly 100% of the time
- ✅ No crashes when using sessions
- ✅ Zero deprecation warnings in build
- ✅ Memory usage reduced by 20%+
- ✅ Startup time < 1 second
- ✅ User can save/restore sessions with 3 clicks or less
