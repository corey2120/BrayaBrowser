- 🍪 **Session Persistence** - Stay logged into your favorite websites

## ✨ What's New

### 1. Session Persistence (Stay Logged In!)
Users can now stay logged into websites after closing and reopening the browser.

**What's Persisted:**
- 🍪 Cookies (stored in SQLite)
- 💾 localStorage data
- 🗄️ IndexedDB databases
- 🔐 Login sessions

**Implementation:**
- WebKitCookieManager with persistent SQLite storage
- Policy: No third-party cookies (privacy-first)
- Automatic directory structure creation
- Location: `~/.local/share/braya-browser/`

**User Impact:** No more re-logging into Gmail, GitHub, Twitter, etc.!

---

### 2. Ad-Blocker Shield UI Redesign

The ad-blocker now has a clean, intuitive interface:

**Shield Button:**
- 🛡️ Blue emoji when enabled
- 🛡️ Grey emoji when disabled
- Click opens popover with toggle switch
- "Settings..." button for advanced configuration

**Popover Contents:**
- Enable/Disable toggle switch
- Direct link to settings page
- Info text about what it blocks

**No More Clutter:**
- Removed statistics from UI (kept internal tracking)
- Simplified settings page
- Focus on core functionality

### 3. Bookmarks Import/Export

Import bookmarks from Chrome, Firefox, or any browser that supports standard formats.

**Supported Formats:**
- **HTML** (Netscape Bookmark File Format)
  - Import from Chrome/Firefox/Safari
  - Export for backup or migration
- **JSON** (Braya Browser format)
  - Native format with full metadata
  - Folder structure preserved

**How to Use:**
1. Open Bookmarks Manager (Ctrl+Shift+O)
2. Click Import/Export buttons
3. Select file format
4. Choose file location

**Technical Details:**
- Proper HTML escaping/unescaping
- Folder hierarchy support
- UTF-8 encoding
- Duplicate handling

### 4. Navigation Button Safety

Fixed crashes when using navigation controls:

**What Was Fixed:**
- Back button bounds checking
- Forward button null pointer checks
- Home button validation
- Reload button safety

**Technical Implementation:**
- Added bounds checking: `activeTabIndex >= 0 && activeTabIndex < tabs.size()`
- Null pointer validation for WebView
- WEBKIT_IS_WEB_VIEW checks before operations
- Window pointer validation

**User Impact:** No more crashes when navigating!

## 🔧 Technical Improvements (from v1.0.7)

These were implemented in v1.0.7 but carry forward to v1.0.8:

### Per-Tab UserContentManager
- **Problem:** SIGABRT crashes when opening 6-8+ tabs
- **Solution:** Each tab gets its own UserContentManager
- **Impact:** Unlimited tabs without handler name conflicts

### 40x Faster Tab Creation
- **Before:** 2000ms per tab (password manager + extension setup)
- **After:** 30-50ms per tab
- **How:** Deferred setup using `g_idle_add()`
- **Impact:** Instant tab opening, setup happens in background

### Memory Safety Fixes
- Fixed use-after-free in deferred setup
- Fixed out-of-bounds array access in popup window creation
- Fixed PopupMessageData memory leak using `g_signal_connect_data`
- Added NULL check in `onCreateNewWindow`
- Fixed g_object_unref balance in password manager

### WebKit Ad-Blocker Implementation
- **WebKitUserContentFilterStore** for disk-cached filters
- Async filter compilation and loading
- Improved blocking rules (specific domains vs broad patterns)
- Changed from `.*ads.*` to `.*doubleclick\.net.*` etc.

## 🔮 Known Issues

### White Flash on New Tabs
- **Status:** Intentional behavior
- **Reason:** Setting dark background breaks website readability
- **Impact:** Brief white flash when opening tabs
- **Workaround:** None (WebKit default behavior, same as Safari)
- **Note:** We tried fixing this but it caused worse problems

### Test Extension Loading Error
- **Status:** Harmless warning
- **Message:** "ERROR: Extension directory not found: ./extensions/hello-world"
- **Impact:** None - just a leftover reference
