# Braya Browser v1.1 Roadmap — "Polish & Power"

**Target:** Major UI polish, settings overhaul, and core feature gaps  
**Vertical tab bar:** Untouched — it stays exactly as-is

---

## Priority 1 — Settings UI Overhaul

The settings window is ~2600 lines of hand-rolled GTK4 widgets. The goal is a clean,
GNOME-style preferences window that feels native and is easy to extend.

### Changes
- [ ] Replace custom nav sidebar with proper `AdwPreferencesWindow` (or GTK4 equivalent)
  - Each section becomes a `AdwPreferencesPage` with `AdwPreferencesGroup` rows
- [ ] Standardize all rows — every toggle uses a consistent layout (label left, control right)
- [ ] Remove the hero/shortcut bar at the top — visual clutter
- [ ] Built-in settings search — find any option instantly
- [ ] Auto-save on change — no more global Apply button
- [ ] Live theme preview — clicking a theme preset applies it immediately
- [ ] Consolidate ad-blocker settings into one "Privacy & Security" page
  - Filter list management (enable/disable EasyList, EasyPrivacy, uBlock filters)
  - Per-list last-updated timestamp
  - Manual refresh button
- [ ] Per-section visual grouping — clear separation between setting groups
- [ ] Keyboard navigation throughout the settings window

---

## Priority 2 — New Tab Page

Replace the current `home.html` with a proper native GTK4 widget page.

- [ ] Clock / date display
- [ ] Inline search bar (uses default search engine)
- [ ] Speed dial — top visited sites pulled from history, with favicons
- [ ] Recent bookmarks strip
- [ ] Clean, minimal layout that respects the active theme (dark/light/industrial)

---

## Priority 3 — Command Palette

`Ctrl+Space` opens a floating search/command overlay.

- [ ] Type to navigate to a URL or search
- [ ] Type to find and open a bookmark
- [ ] Type to run browser commands (New Tab, Settings, Toggle Dark Mode, etc.)
- [ ] History search inline
- [ ] Keyboard-only navigation (arrow keys + Enter)
- [ ] Dismisses on Escape or click-outside

---

## Priority 4 — Private / Incognito Mode

`Ctrl+Shift+N` opens a private window.

- [ ] Separate `WebKitWebContext` with no disk persistence
- [ ] Visual indicator — purple accent on sidebar or window badge
- [ ] No history, no cookie persistence, no cache written to disk
- [ ] Password manager disabled in private windows
- [ ] "You are browsing privately" indicator on new tab page

---

## Priority 5 — Per-Site Settings & Zoom

- [ ] Right-click tab → "Site Settings" popover
  - JavaScript on/off
  - Cookie policy (allow / block / session-only)
  - Ad-blocker whitelist toggle
- [ ] Zoom controls — `Ctrl++` / `Ctrl+-` / `Ctrl+0`
- [ ] Zoom level shown in URL bar when not 100%
- [ ] Zoom persisted per-domain

---

## Priority 6 — Reliability & Performance

- [ ] Ad-blocker filter auto-update — weekly background refresh
  - UI shows "Updated X days ago" per filter list
- [ ] Crash recovery — web process crash shows a proper "Tab Crashed" page with Reload button
  - Currently shows a blank tab
- [ ] Session restore — actually save and restore open tabs on quit/crash
  - The setting toggle exists but needs full implementation
- [ ] GTK4 deprecation audit — fix deprecated API calls before they become errors
- [ ] Memory indicator improvement — use `/proc/self/smaps` for accurate per-process memory

---

## Build Order

| # | Feature | Why first |
|---|---|---|
| 1 | Settings UI rewrite | Biggest visible impact, sets v1.1 tone |
| 2 | New Tab page | Second thing users see constantly |
| 3 | Command palette | Self-contained, high power-user value |
| 4 | Private mode | Commonly expected, builds on WebKit context work |
| 5 | Per-site settings + zoom | Polish layer on top of existing tab system |
| 6 | Filter auto-update + crash recovery | Reliability / maintenance |

---

## What Stays the Same

- **Vertical tab bar** — untouched, it's a core differentiator
- **Extension system** — no rewrite in v1.1
- **Password manager** — no breaking changes, only additive
- **Ad-blocker engine** — the uBlock filter backend stays, only the settings UI changes
- **Themes** — dark/light/industrial remain, live preview is additive
